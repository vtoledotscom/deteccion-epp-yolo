# Instrucciones para el agente IA

## Proyecto
Sistema de detección de EPP.

## Stack
- Laravel
- PHP
- Blade
- MySQL/MariaDB
- Python
- YOLO
- Ubuntu Server
- Apache

## Reglas obligatorias
- No modificar archivos sin autorización explícita.
- Antes de cambiar código, explicar:
  - archivo afectado
  - motivo del cambio
  - impacto esperado
- No tocar archivos .env ni credenciales.
- No modificar migraciones existentes sin autorización.
- No cambiar nombres de rutas sin avisar.
- No cambiar estructura de base de datos sin justificar.
- No eliminar código funcional sin explicar.
- Mantener compatibilidad con producción.
- Priorizar cambios mínimos y seguros.
- Después de modificar, indicar comandos de prueba.
- Si el cambio falla, indicar rollback.

## Flujo obligatorio
1. Analizar.
2. Explicar causa o mejora.
3. Proponer cambio.
4. Esperar autorización.
5. Modificar archivos.
6. Mostrar resumen.
7. Indicar pruebas.
